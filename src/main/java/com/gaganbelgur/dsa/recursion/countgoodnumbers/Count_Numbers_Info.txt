A digit string is a string consisting only of the digits '0' through '9'. It may contain leading zeros.

1. Why Square every iteration?
In binary exponentiation, we square the base at each iteration to efficiently compute powers. This is because each
bit in the binary representation of the exponent corresponds to a power of 2. By squaring the base,
we can quickly compute higher powers without having to multiply the base repeatedly.

base = 5
After one Squaring: 5^2 = 25
After two Squaring: 5^4 = 625
After three Squaring: 5^8 = 390625

Each squaring operation effectively doubles the exponent, allowing us to reach the desired power more quickly.

2. Why divide the exponent by 2?
In binary exponentiation, we divide the exponent by 2 at each iteration to process the bits of the exponent from
least significant to most significant. This allows us to determine whether to include the current power of the base
in the final result based on whether the current bit is 1 or 0. By halving the exponent, we can efficiently traverse
through its binary representation and build the result step by step.

Example: 13 = (1101) in binary
- Start with exponent = 13 (binary 1101)

- Current bit (LSB) is 1 → include 5^1 in the result
- Divide exponent by 2 → exponent = 6 (binary 110)

- Current bit is 0 → do not include 5^2 in the result
- Divide exponent by 2 → exponent = 3 (binary 11)

- Current bit is 1 → include 5^4 in the result
- Divide exponent by 2 → exponent = 1 (binary 1)

- Current bit is 1 → include 5^8 in the result
- Divide exponent by 2 → exponent = 0 (binary 0)

This process continues until the exponent reaches 0, at which point we have included all necessary powers
of the base in the final result.

3. Why take modulo 10^9 + 7?
Taking modulo 10^9 + 7 is a common practice in competitive programming and algorithm design
to prevent integer overflow and to keep numbers manageable. By taking results modulo 10^9 + 7, we ensure that our calculations
remain within the bounds of standard integer types.
